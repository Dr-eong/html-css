What is Javascript?

JavaScript (줄여서 "JS")는 HTML 문서에 적용될 때, 웹사이트상에서 동적 상호작용성을 제공할 수 있는 완전한 동적 프로그래밍 언어(dynamic programming language)입니다. 이것은 Mozilla 프로젝트, Mozilla 재단, 그리고 Mozilla 법인의 공동 창설자인 Brendan Eich 에 의해 만들어졌습니다.

자바스크립트는 믿을 수 없을 정도로 다재다능합니다. 캐러셀 기능(이미지를 차례대로 바꿔볼 수 있는 기능)을 가진 이미지 갤러리, 변화하는 레이아웃, 그리고 버튼이 클릭될 때의 반응과 같은 간단한 것부터 시작할 것입니다. 경험이 더 쌓이면 게임이나 움직이는 2D 및 3D 그래픽, 포괄적인 데이터베이스 지향적인 앱과 더 많은 것을 만들 수 있을 것입니다!

자바스크립트 그 자체는 상당히 작지만 아주 유연합니다. 개발자들은 코어 자바스크립트 언어(core JavaScript language) 위에서 동작하는 많은 다양한 도구를 개발해왔는데, 이를 이용하면 최소한의 수고로 엄청나게 많은 확장 기능을 사용할 수 있습니다. 여기에는 다음과 같은 것들이 포함됩니다:

  • 브라우저 응용 프로그래밍 인터페이스 (APIs) — 브라우저에 내장된 API로 HTML을 동적으로 생성하고 CSS 스타일을 설정하거나, 사용자의 웹캠으로부터 비디오 스트림을 수집하거나 조작하는 것, 또는 3D 그래픽이나 오디오 샘플을 생성하는 것과 같은 다양한 기능을 제공합니다.
  • 제3자 (third-party) API를 활용해 개발자는 트위터나 페이스북 같은 다른 컨텐츠 공급사부터 제공되는 기능을 자신의 사이트에 통합할 수 있습니다.
  • 써드파티 프레임워크와 라이브러리를 여러분의 HTML에 적용함으로써 사이트와 어플리케이션을 빠르게 구축할 수 있습니다.

위의 내용은 자바스크립트의 매우 일부분입니다. 자바스크립트의 더 많은 내용이 궁금하다면 이곳 을 방문해보세요!

Data Type

자바스크립트의 데이터 종류는 크게 3가지로 분류됩니다. 문자를 나타내는 Stiring, 숫자를 나타내는 Number, 참과 거짓을 나타내는 Boolean이 그것입니다.

지금부터 이것들에 대해 알아보겠습니다.

  1. String : 자바스크립트 작성 시에 따옴표 안에 들어가는 문자를 의미한다. const name = "kim";
  2. Number : 따옴표를 사용하지 않고 작성된 숫자들을 의미한다. let age = 23;
  3. Boolean : 자바스크립트에서 부울 값은 TRUE 또는 FALSE일 수 있는 값이다. 무언가에 대해 "예" 또는 "아니오"를 알아야 한다면, 부울 함수를 사용하고 싶을 것입니다. 그것은 매우 간단하게 들리지만, 부울은 자바스크립트 프로그래밍에서 항상 사용되며, 매우 유용하다. "켜짐" 또는 "꺼짐", "예" 또는 "아니오", "참" 또는 "거짓"이어야 하거나 일시적인 목적이 있는 모든 것은 일반적으로 부울에게 적합합니다. true / false;
Variable 


      const a = value;("문자", 숫자, 함수, 불리안 등)
      let b = value;(수정 가능성이 존재하는 값)

variable(변수)란 number나 string과 같은 값의 컨테이너입니다. 변수 안에는 복잡한 데이터나 함수 등 어떤 것도 할당될 수 있습니다.
위에 코드를 보면 "const"와 "let"을 볼 수 있습니다. "const"는 할당된 값이 변하지 않는 변수를 선언할 때 사용됩니다. "let"은 할당된 값이 수정 가능성이 있거나 변해야하는 값일 경우에 선언합니다. 또한 "="가 있는데 이것은 "할당연산자"로서 우측의 변수에 좌측의 값을 할당하는 역할을 합니다.

아래 리스트는 변수 이름에 대한 규칙입니다.

  • 일반적으로 라틴문자와 알파벳 소문자, 대문자, 밑줄 문자 사용
  • 변수 이름의 시작부분에 밑줄(_) 사용 금지
  • 변수 이름의 시작부분에 숫자 사용금지
  • “lower camel case” : 첫단어의 시작은 소문자, 그 다음 단어의 시작은 대문자. (ex; lastResult)
  • 변수 이름은 저장된 값을 설명하는 자기설명적이어야 한다
  • 변수는 대소문자를 구분함. (myage ≠ myAge)
  • 구문을 작성하는 예약어로 변수 이름 작성 금지 (ex; var, let, for, function…)
Array 


      const fruits = ["apple", "banana", "pineapple"];

배열은 하나의 변수명 아래에 데이터 아이템의 리스트를 저장하는 간편한 방법입니다. 만약 배열이란 개념이 존재하지 않는다면 모든 값을 별도의 변수에 저장해야 할 것이고, 각각의 값을 출력하고 계산하는 코드를 추가해야 할 것 입니다. 이런 방식은 시간도 더 들고 비효율적이며 오류가 발생하기 쉽습니다.

배열의 작성법은 대괄호[]로 구성되며 쉼표(,)로 구분된 항목을 포함합니다. 이때 대괄호 안에 들어갈 수 있는 값은 문자, 숫자, 불리안.. 등 모든 값이 해당됩니다.

Function 


      function functionName() {
       functionBody;
      }

코딩에 있어서 필수적인 요소인 함수는 쉽게 말해서 반복해서 사용할 수 있는 코드조각을 의미합니다. 사용방법은 functionName(); 입니다. 거창한 기술이나 코드가 필요한 게 아니라 함수명에 소괄호만 작성하면 끝입니다. functionBody에는 실행하고자 하는 어떤 것도 넣을 수 있습니다.

함수의 작성법은 "function 함수이름() {};"입니다.
함수이름 옆 소괄호는 매개변수의 자리입니다. 중괄호 안에는 함수를 실행했을 때 작동하고 싶은 코드가 위치합니다. 아래 내용은 함수 선언 시 주의해야 할 부분입니다.

  • 호출 인자 누락
  • 호출 인자 속성 누락
  • API 호출 시 인자 타입 오류
  • API 호출 시 인자 개수 오류
IF

어떤 프로그래밍 언어든 코드는 의사 결정을 내리고 입력 내용에 따라 작업을 수행해야 합니다. 예를 들어 게임에서 플레이어의 생명 수치가 0이면 게임이 종료됩니다. 날씨 앱에서는 아침에 해가 뜬 그림을 보여주고 밤에는 달과 별을 보여줍니다. 이 문서에서는 JavaScript에서 조건문이 작동하는 방법을 살펴 보겠습니다. 


      if (조건) {
       만약 조건(condition)이 참일 경우 실행할 코드
      } else {
       대신 실행할 다른 코드
      }
     

      if (조건) {
       만약 조건(condition)이 참일 경우 실행할 코드
      } 
       
      대신 실행할 다른 코드

가장 일반적인 조건문은 if...else 조건문입니다. 조건문의 실행 순서는 아래와 같습니다.

  1. 키워드 if 뒤에 괄호가 옵니다.
  2. 시험할 조건은 괄호 안에 위치합니다 (전형적으로 "이 값은 다른 값보다 큰가?", 또는 "이 값은 존재하는가?"). 이 조건은 마지막 모듈에서 논의했던 비교 연산자(comparison operators)를 사용할 것이고 true나 false를 리턴합니다.
  3. 내부의 중괄호 안에 코드가 있습니다. — 이것은 우리가 좋아하는 어떤 코드든 될 수 있고, 오직 조건이 true를 반환하는 경우에만 실행됩니다.
  4. 키워드 else
  5. 또 다른 중괄호 안에 더 많은 코드가 있습니다. — 이것은 우리가 좋아하는 어떤 코드든 될 수 있고, 오직 조건이 true가 아닌 경우에만 실행됩니다. — 또는 다른 말로 하자면, 조건이 false인 경우에만 실행됩니다.

이 코드는 사람이 읽을 수 있습니다. — 이것은 "만약 조건이 true면, 코드 A를 실행하고, 아니면 코드 B를 실행한다." 라고 말합니다. 두번째 코드 예시를 보면 첫번째 코드 예시처럼 반드시 else와 두 번째 중괄호를 포함하지 않아도 된다는 것을 주목해야 합니다.
그러나 여기서 조심해야 할 점은 코드의 두 번째 블록은 조건문에 의해서 제어되지 않아서, 조건이 true나 false를 리턴하는 것에 관계없이 항상 동작한다는 것 입니다. 이것이 반드시 나쁜 것은 아니지만, 여러분이 원하는 것이 아닐 지도 모릅니다.
